Legibilidade

de uma Linguagem de Programação

A legibilidade é um critério de avaliação que diz respeito à facilidade com que o código pode ser lido e compreendido por humanos. Este entendimento é importante para a manutenção dos programas.

A legibilidade pode ser avaliada por meio das seguintes características:

Simplicidade Global:

Uma linguagem de programação simples é mais fácil de aprender e usar, o que contribui para sua legibilidade. A simplicidade global é alcançada por meio de uma combinação de fatores, incluindo:

Um pequeno conjunto de conceitos fundamentais: Uma linguagem com um pequeno conjunto de conceitos fundamentais é mais fácil de entender e lembrar. Uma sintaxe consistente: Uma sintaxe consistente facilita a visualização da estrutura do código. Um número limitado de recursos: Um número limitado de recursos reduz a complexidade da linguagem.

Ortogonalidade:

A ortogonalidade é uma propriedade de uma linguagem que permite que os recursos sejam combinados de maneiras diferentes para obter diferentes efeitos. Uma linguagem ortogonal é mais flexível e expressiva, o que contribui para sua legibilidade. Por exemplo, a linguagem C++ é considerada uma linguagem ortogonal porque permite que os tipos de dados sejam combinados para formar tipos de dados compostos.

A ortogonalidade em programação significa que as funcionalidades do sistema ou da linguagem de programação são independentes umas das outras, de modo que uma mudança em uma parte do sistema não afeta outras partes. Isso torna o sistema mais flexível e menos propenso a bugs. Já a ortogonalidade em uma linguagem de programação significa que um conjunto relativamente pequeno de construções primitivas pode ser combinado a um número relativamente pequeno de formas para construir as estruturas de controle e de dados da linguagem.

Instruções de controle:

As instruções de controle são usadas para alterar o fluxo de execução de um programa. Instruções de controle claras e concisas contribuem para a legibilidade do código. Por exemplo, a instrução if-else é uma instrução de controle clara e concisa que pode ser usada para implementar uma condição. Elas permitem que um programa tome decisões e execute diferentes ações com base em condições ou itere sobre um conjunto de dados.

Tipos de dados e estruturas:

Os tipos de dados e estruturas são usados para representar dados no programa. Tipos de dados e estruturas bem projetados contribuem para a legibilidade do código, tornando-o mais fácil de entender e manter. Por exemplo, a linguagem Java fornece um conjunto de tipos de dados bem projetados, como int, float e String. As estruturas de dados são maneiras de organizar e armazenar esses tipos de dados, como arrays, listas, pilhas, filas, árvores, entre outros.

Sintaxe:

A sintaxe é o conjunto de regras que definem como o código é escrito e interpretado. Uma sintaxe clara e concisa contribui para a legibilidade do código, tornando-o mais fácil de ler e entender. Por exemplo, a linguagem Python tem uma sintaxe clara e concisa, com poucas regras complexas.
