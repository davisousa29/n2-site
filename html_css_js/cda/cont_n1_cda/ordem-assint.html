Ordem assintótica (notações)

Notação Assintótica é uma linguagem que nos permite analisar o tempo de execução de um algoritmo através da identificação de seu comportamento com o crescimento da entrada oferecida. Isso também é conhecido como taxa de crescimento do algoritmo.

Descrevemos como Notação Assintótica identifica o comportamento de um algoritmo à medida que o tamanho da entrada cresce. Para dado algoritmo f, com entrada de tamanho n, você terá tempo de execução f(n). Isto resulta em um gráfico onde a coordenada Y é o tempo de execução, a coordenada X representa o tamanho da entrada e os pontos representam o tempo de execução para dado tamanho de entrada.

Um algoritmo pode ser representado nas formas de Melhor Caso, Pior Caso ou Caso Médio. A maneira mais comum de analisar um algoritmo é pelo Pior Caso.

Tipos de funções, limites e simplificação:

- Função Logarítmica - log n
- Função Linear - an + b
- Função Quadrática - an2+bn+c
- Função Polinomial - anz+...+an2+an1+an0, onde "z" é uma constante
- Função Exponencial - an, onde "a" é alguma constante

Estas são as funções básicas de crescimento usadas em várias notações. Uma nota extremamente importante para notações é tentar usar os termos mais simples. Isto significa descartar constantes e termos de ordem baixa, pois quando o tamanho da entrada cresce para o infinito (limites matemáticos), os termos de ordem mais baixa e constantes tornam-se irrelevantes. Por exemplo, se você tiver uma constante muito grande, 29001, a simplificação não afetará sua notação.

Big O

A notação Big O é usada para descrever o limite superior assintótico da complexidade de um algoritmo. Isso significa que ela descreve o número máximo de operações que um algoritmo pode realizar, independentemente do tamanho dos dados de entrada.

Por exemplo, o algoritmo de ordenação quicksort tem uma complexidade de Big Omega(n log(n)). Isso significa que, no melhor caso, o algoritmo precisará executar n log(n) operações para ordenar uma lista de n elementos.

Big Theta

A notação Big Theta é usada para descrever a complexidade exata de um algoritmo. Isso significa que ela descreve o número de operações que um algoritmo realiza, independentemente do tamanho dos dados de entrada.

Por exemplo, o algoritmo de ordenação mergesort tem uma complexidade de Big Theta(n log(n)). Isso significa que o algoritmo precisará executar n log(n) operações, independentemente do tamanho dos dados de entrada

Diferença entre Big O, Big Omega e Big Theta

O Big Omega é útil para descrever algoritmos que são muito eficientes, pois ela garante que o algoritmo realizará um número mínimo de operações, independentemente do tamanho dos dados de entrada.

O A Big Theta é útil para descrever algoritmos que possuem uma complexidade bem definida, independentemente do tamanho dos dados de entrada.
